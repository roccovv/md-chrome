# 天气功能配置指南

## 功能特性

✅ **实时天气** - 显示当前温度、天气状况、体感温度  
✅ **自动刷新** - 每5分钟自动更新天气数据  
✅ **15天预报** - 点击日历图标查看未来15天天气  
✅ **自定义城市** - 支持修改显示城市（默认：深圳）  
✅ **精准数据** - 使用和风天气 API，数据准确可靠

## 如何获取免费 API Key

### 1. 注册和风天气账号

访问：https://dev.qweather.com/

点击右上角「注册」或「登录」

### 2. 创建项目（重要！）

登录后进入控制台：https://console.qweather.com/

- 点击「项目管理」
- 点击「创建项目」
- 填写项目信息：
  - **项目名称**：MD Chrome 天气插件（随意填写）
  - **选择KEY类型**：⚠️ **必须选择「Web API」**
  - 点击「创建」

**⚠️ 关键步骤：配置白名单**

创建项目后，需要配置允许访问的域名：

1. 在项目列表中，点击你创建的项目
2. 找到「服务状态」或「设置」
3. 在「白名单」或「Allowed Origins」中添加：
   ```
   chrome-extension://*
   ```
   或者留空（表示允许所有来源，仅开发测试用）

**如果你的控制台没有白名单设置**，说明你的 API Key 是免费开发版，可能有限制。建议：
- 检查 KEY 类型是否为「Web API」
- 查看是否有「数量限制」或「请求限制」
- 确认账号是否已验证（邮箱验证）

### 3. 获取 API Key

创建成功后，在项目列表中可以看到你的 **KEY**（一串字符串）

复制这个 KEY

### 4. 配置到项目中

打开文件：`src/components/Weather.tsx`

找到第 29 行：

```typescript
const QWEATHER_KEY = 'YOUR_API_KEY_HERE';
```

将 `YOUR_API_KEY_HERE` 替换为你的 API Key：

```typescript
const QWEATHER_KEY = '你复制的KEY';
```

### 5. 重新构建

```bash
npm run build
```

## 故障排查

### 天气显示"加载中..."

**原因**：API Key 未配置或配置错误

**解决**：
1. 检查 `Weather.tsx` 中的 `QWEATHER_KEY` 是否正确
2. 检查网络连接
3. 打开浏览器控制台查看错误信息

### 找不到城市

**原因**：城市名称输入错误

**解决**：
1. 使用标准城市名称（如：深圳、北京）
2. 不要加"市"字（错误：深圳市 ✅ 正确：深圳）
3. 使用中文名称

### API 请求失败（403 Forbidden）

**原因**：API Key 权限问题

**解决方案**：

1. **检查 KEY 类型**
   - 登录 https://console.qweather.com/
   - 查看你的项目
   - 确认 KEY 类型是「Web API」而不是「iOS」或「Android」

2. **配置白名单**（如果有此选项）
   - 在项目设置中找到「白名单」或「Allowed Origins」
   - 添加：`chrome-extension://*`
   - 或者暂时留空（允许所有来源）

3. **重新创建 API Key**
   - 如果以上都不行，删除旧项目
   - 重新创建一个新的「Web API」项目
   - 确保账号已验证（邮箱验证）

4. **使用备用方案**
   - 如果和风天气一直 403，可以考虑使用其他免费天气 API
   - 例如：OpenWeatherMap、高德天气等
